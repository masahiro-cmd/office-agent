# セキュリティポリシー

## 設計上のセキュリティ制約

### 1. ネットワーク通信

- LLM バックエンドは **ローカルホストのみ** と通信します（Ollama: `localhost:11434`, llama.cpp: `localhost:8080`）
- 外部インターネットへの接続は一切行いません
- LLM プロンプトに外部 URL や環境変数を含めません

### 2. ToolRegistry ホワイトリスト

許可されたツールのみ呼び出し可能です：

```
create_docx
create_xlsx
create_pptx
generate_vba
read_local_text_file
```

未登録のツール名を指定した場合、`PermissionError` が発生します。
LLM の出力がどのようなツール名を要求しても、ホワイトリスト外は実行されません。

### 3. ファイル読み取りサンドボックス

`read_local_text_file` は `ALLOWED_READ_DIRS` 設定で許可されたディレクトリ内のファイルのみ読み取り可能です。

- デフォルト許可: `./`, `/tmp`
- パストラバーサル（`../`）を含むパスは拒否されます
- シンボリックリンクの解決後のパスで検証します

### 4. VBA 生成

- VBA コードの**生成のみ**を行います
- 生成した VBA コードを**実行する機能はありません**
- 出力は `.bas` ファイルまたは `.md` ファイルとして保存されます

### 5. 更新システム

- 更新パッケージは SHA-256 ハッシュで署名されます
- 適用前に必ずハッシュ検証が実施されます
- 検証に失敗した更新パッケージは適用されません
- 署名検証には Python 標準ライブラリ (`hashlib`) のみを使用します

### 6. ログ

- ログには機密情報（パスワード、個人情報等）を含めません
- `logs/` ディレクトリは `.gitignore` に追加されており、リポジトリに含まれません

---

## 脆弱性報告

セキュリティ上の問題を発見した場合は、Issue ではなく直接メンテナに報告してください。

---

## 既知の制限事項

- LLM の出力は完全には制御できません。生成された文書の内容を必ず確認してください
- JSON スキーマ検証は構造を保証しますが、内容の安全性は保証しません
- 大量データの処理時にメモリ使用量が増加する場合があります
